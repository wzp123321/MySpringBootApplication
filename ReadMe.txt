前端：
1.开发语言：H5 CSS JS Jquery  vue框架  BootstrapVue框架
2.开发工具：vscode
3.端口号：8182
4.登录注册等验证由前端实现
5.登陆拦截也由前端实现
6.前端封装ajax请求
7.登录注册的对话框用animate动画特效
    登录时如果选中记住密码：对用户名密码进行加密crypto-js
8.样式用less(Less 是一门 CSS 预处理语言,)
9.图形化验证码(canvas)




后端：
1.开发语言：Java SpringBoot框架
2.开发工具：IDEA
3.端口号：9898
4.前后端分离
  跨域问题由后端实现
          @CrossOrigin// 解决前后端分离的跨域问题
          以及配置类 MyConfiguration
5.对于登录的处理：
      1.为用户随机生成一个盐(32位)
      2.将用户输入的密码与对应的盐通过算法进行加密
      3.登录时先获取用户自己的盐和加密后的密码然后调用方法进行验证
6.token后端生成然后进行校验
    JWT(json web token)生成token 然后拦截器中掉方法解析token;如果解析成功说明token有效,否则无效
7. @ControllerAdvice注解处理异常-----捕获请求过程中产生的异常:(参数格式错误/必填参数为空。。。)
8.自定义拦截器：每次请求之前校验token(除用户登录注册外)
9.文件上传：在yml文件中配置文件存储位置
10.数据源druid:
   DRUID是阿里巴巴开源平台上一个数据库连接池实现，它结合了C3P0、DBCP、PROXOOL等DB池的优点，同时加入了日志监控，可以很好的监控DB池连接和SQL的执行情况，
   可以说是针对监控而生的DB连接池(据说是目前最好的连接池,不知道速度有没有BoneCP快)。






数据库：

接口命名规则：
模块/界面/功能
eg：usermodule/login/select  用户模块/登录界面/查询功能



状态码分配:
    1.成功： 200
    查询失败/删除失败/修改失败：301
    密码错误:302
    用户不存在/用户名被占用:303
    请求方法校验失败:400
    必填参数为空/数字格式错误/请求参数格式不正确：401
    token校验失败(已失效或者不存在)：403
    服务器内部错误：500



    业务失败：401
